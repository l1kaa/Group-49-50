1) გაიხსენეთ რა არის git და რა არის github; რა განსხვავებაა მათ შორის, რა არის მათი დანიშნულება.

git არის control სისტემა, რომელიც შექმნილია იმისთვის, რომ კონნკრეტული პროექტის შექმნის (ანუ development-ის) დროს კომპიუტერში შევინახოთ პროექტის კოდში შეტანილი ცვლილებები. 

ხოლო Github არის პლათფორმა, სადაც Git-ის დახმარებით შეგვიძლია ჩვენს მიერ შექმნილი ინდივიდუალური თუ კოლაბორაციული პროექტები ავტვირთოთ. Github საუკეთესო გარანტიაა იმისთვის, რომ ჩვენი კოდი არ დაიკარგოს.

Git-ს ბევრი დადებითი მხარე აქვს, მაგ:
• ვერსიების კონტროლი
• კოდის ორგანიზება/გადანაწილება
• კოლაბორაციული პროექტების აგება
• Git-ის Merging და Pull request-ები
• და ა.შ


2) რა არის commit და რისთვის გვჭირდება Gitში

commit არის ერთგვარი 'კომენტარი'. უფრო დეტალურად რომ ავხსნათ: commit საშუალებას გვაძლევს Git-ის გამოყენებით ჩვენს კოდში შეტანილი ცვლილებები დავასათაუროთ, რომ არ გაგვიჭირდეს ვერსიების კონტროლი.

3) რა განსხვავებაა main branchსა და სხვა დანარჩენ branchებს შორის? და საერთოდ რატომ ვიყენებთ branch-ებს?

main branch - (როგორც ამას სახელიდან გამომდინარე ვხვდებით -->) მთავარი 'მისამართია', სადაც შეგვიძლია ჩვენი კოდი ავტვირთოთ.

სხვა დანარჩენი ბრანჩებისგან განსხვავებით იგი ყველაზე მნიშვნელოვან მისამართს წარმოადგენს. დანარჩენ branch_ებზე შესაძლოა ჩვენი პროექტის პატარ-პატარა ნაწილაკები ან სხვა დეტალები ავტვირთოთ.

branch-ებს კი ვიყენებთ იმისთვის, რომ პროექტის ნაწილაკები განვაცალკევოთ, ან დეველოპერების გუნდთან ერთად მუშაობის დროს - თითოეულმა დეველოპერმა საკუთარი branch-იდან ატვირთოს კოლაბორაციული პროექტიდან თავისი წილი კოდი.

4) გით ჰაბიდან შექმენით ახალი რეპოზიტორია. 
შექმენით ახალი ლოკალური რეპო თქვენს pc ზე და დააკავშირეთ გითჰაბთან.
შექმენით ფაილი hello.txt და ჩაწერეთ მასში „Hello Git!“ ან რაც თქვენ გინდათ.
თუ ახალი ვერსისს გიტი გაქვთ მაშინ პირდავირვე გექნებათ main ბრენჩი თუ ძველი ვერსიის მაშინ გამოიყენეთ კომანდა git branch -M main
დაამატეთ commit (git add . და git commit -m "First Commit In This Repo").
დაპუშეთ ეს კომიტი main branch ზე

5) ახლად შექმნილ რეპო ში შექმენით ახალი branch ნებისმიერი სახელით და შეცვალეთ hello.txt ფაილის შიგთავსი(დაამატეთ/გამოაკელით ტექსტი)
დააკომიტეთ და დაპუშეთ ეს ცვლილებები ახალ branch_ში
